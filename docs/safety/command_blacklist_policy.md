# 高危命令黑名单策略

## 目标
定义默认禁止执行的高危命令模式，避免系统级不可逆破坏。

## 默认禁止命令
- rm -rf /
- rm -rf /*
- mkfs
- fdisk 写盘操作
- dd 写入磁盘设备
- shutdown 或 reboot（未授权场景）
- :(){ :|:& };: 炸弹脚本

## 默认禁止路径
- /
- /boot
- /etc
- /usr
- /bin
- /sbin
- /lib
- /var/lib

## 规则匹配建议
1. 先规范化命令文本（去多余空白、统一引号）。
2. 对危险关键字做精确匹配与正则匹配组合。
3. 同时做路径风险判断，避免通过变量绕过。

## 拒绝反馈模板
- 检测到高危命令模式，已拒绝执行。
- 原因：命令可能导致系统不可恢复损坏。
- 建议：改用查询或模拟命令，必要时走管理员审批。

## 误报处理
- 对教学环境和沙箱环境可配置例外名单。
- 例外必须带有效期、审批人、操作范围。
