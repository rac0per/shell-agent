# 批量权限修复 SOP

## 任务目标
修复目录权限异常，确保服务可访问且不过度授权。

## 前置检查
- 确认目标目录范围。
- 确认期望权限模板（文件 644，目录 755 等）。
- 确认服务运行用户。

## 操作步骤
### 第 1 步：查看当前权限
find /data/app -maxdepth 2 -exec ls -ld {} \; | head -50

### 第 2 步：修复目录权限
find /data/app -type d -exec chmod 755 {} \;

### 第 3 步：修复文件权限
find /data/app -type f -exec chmod 644 {} \;

### 第 4 步：修复属主
chown -R appuser:appgroup /data/app

## 风险点
- 过宽权限会引入安全风险。
- 过窄权限会导致服务启动失败。

## 回滚与应急
- 变更前导出权限快照。
- 关键目录先小范围试运行再全量执行。

## 对应自然语言问法
- 统一修复这个目录权限，别影响服务运行。
