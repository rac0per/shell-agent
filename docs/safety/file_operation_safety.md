# 文件操作安全规范

## 高风险命令分级
### 一级禁止（默认拒绝）
- rm -rf /
- mkfs
- dd 直接覆盖磁盘设备
- 未审计的 sudo 写操作

### 二级谨慎（需二次确认）
- rm -rf 目录删除
- chmod -R 大范围改权限
- chown -R 大范围改属主
- mv 批量重命名或跨目录迁移

### 三级可执行（建议预演）
- find 或 grep 查询
- du 或 df 容量分析
- cat 或 tail 日志查看

## 预执行三步校验
1. 语法校验：命令可被当前 Shell 正确解析。
2. 风险评估：识别是否涉及删除、覆盖、权限变更。
3. 效果模拟：先用 echo 或仅查询命令输出待处理对象。

## 建议的确认文案
- 该命令将删除 N 个文件，是否继续？
- 该命令将修改目录权限，是否继续？
- 该命令影响系统路径，建议先在测试环境执行。

## 回滚建议
- 删除类操作先转移到回收目录，不直接永久删除。
- 关键配置文件变更前先创建备份副本。
- 批量操作保留前后差异清单。

## 日志记录字段
- 用户自然语言输入。
- 生成命令。
- 风险等级。
- 执行结果与时间戳。
- 会话 ID。
