# 权限控制策略

## 目标
确保 Shell Agent 在最小权限原则下运行，避免越权执行。

## 角色分级
- 访客：仅允许查询命令。
- 操作员：允许低风险写操作（受限目录）。
- 管理员：可执行高风险命令，但必须审批留痕。

## 权限边界
- 默认禁止 sudo。
- 禁止修改系统关键路径。
- 禁止访问未授权业务目录。

## 提权流程
1. 申请：提交命令、目标范围、原因。
2. 审批：管理员确认风险与窗口。
3. 执行：仅在授权时段和范围执行。
4. 复盘：记录结果与异常。

## 临时授权
- 必须包含过期时间。
- 仅授予单次或短时操作。
- 到期自动撤销。

## 审计字段
- 用户身份与角色。
- 授权单号。
- 命令文本与参数。
- 影响对象与结果。
