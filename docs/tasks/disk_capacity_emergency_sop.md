# 磁盘容量应急处理 SOP

## 任务目标
当磁盘使用率过高时快速定位占用热点并执行低风险缓解。

## 前置检查
- 确认告警阈值与当前使用率。
- 确认受影响挂载点。
- 确认关键服务写入路径。

## 操作步骤
### 第 1 步：查看磁盘总体使用
df -h

### 第 2 步：定位大目录
du -sh /* 2>/dev/null | sort -hr | head -20

### 第 3 步：定位大文件
find /var -type f -size +500M 2>/dev/null

### 第 4 步：执行缓解动作（示例：压缩老日志）
find /var/log -type f -name "*.log" -mtime +7 -print0 | xargs -0 gzip

## 风险点
- 直接删除可能影响审计和故障排查。
- 高峰期 I/O 可能进一步恶化性能。

## 回滚与应急
- 优先压缩和迁移，避免直接删除。
- 记录处理前后容量变化用于复盘。

## 对应自然语言问法
- 磁盘快满了，先帮我查哪里占用最大。
- 给我一个低风险的磁盘应急处理步骤。
