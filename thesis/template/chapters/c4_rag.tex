\chapter{领域RAG检索系统}

\section{知识库构建}

\subsection{文档结构设计}

知识库位于\texttt{docs/}目录，按5个类别组织，共38篇文档，覆盖约150种用户意图：

\begin{itemize}
    \item \texttt{commands/}：21篇命令参考卡（tar、chmod、find、grep等）
    \item \texttt{safety/}：8篇安全策略文档
    \item \texttt{tasks/}：6篇SOP流程文档
    \item \texttt{patterns/}：2篇模式文档
    \item \texttt{examples/}：1篇示例文档
\end{itemize}

\subsection{文档增强策略}

为缓解自然语言查询与文档表面形式之间的词汇鸿沟（Vocabulary Gap）问题，对所有文档进行系统性增强，统一添加如下结构：

\begin{enumerate}
    \item \textbf{同义问法}：覆盖口语化、被动语态、中英混杂等多样化表达形式；
    \item \textbf{覆盖失败Query}：将历史检索失败的原始查询直接嵌入文档；
    \item \textbf{文档元信息}：标注类别、意图标识、风险等级和目标Shell类型。
\end{enumerate}

此增强策略将模板查询集的双臂MRR从约0.92提升至\textbf{1.0（满分）}，验证了KB增强对缓解词汇鸿沟的有效性。

\section{混合检索策略}

\subsection{纯向量检索基线}

使用ChromaDB向量数据库配合BAAI/bge-small-zh-v1.5中文嵌入模型进行向量检索，通过余弦相似度计算查询与文档的语义相关度。

\subsection{混合检索与重排}

在纯向量检索基础上，引入关键词权重乘积重排：

$$\text{score}(d) = \text{vector\_score}(d) \times (1.0 + 0.2 \times \text{keyword\_score}(d))$$

其中$\text{vector\_score}$为ChromaDB余弦相似度（归一化至$[0,1]$），$\text{keyword\_score}$为查询分词后对文档的关键词命中率。乘积因子0.2经调参确定，在关键词补强与向量语义之间取得平衡。

\section{类别路由过滤}

实现于\texttt{src/rag\_routing.py}，通过关键词规则将用户查询映射到5个类别之一，在检索时施加类别过滤以缩小候选集。路由优先级（顺序敏感）如下：

\begin{enumerate}
    \item 精确短语预检（避免后续规则误分）
    \item Safety关键词：安全、危险、黑名单等
    \item Tasks关键词：SOP、备份、巡检等
    \item Patterns关键词：bash、zsh、差异等
    \item Examples关键词：例子、示例等
    \item 默认：commands
\end{enumerate}

路由规则经22个测试用例验证，全部通过（含跨类边界case）。

\section{评估集构建}

\subsection{模板评估集}

早期构造38个基于文档名称的模板查询，用于验证KB覆盖完整性。双臂最终均达到MRR=1.0。

\textbf{局限}：与文档标题高度重合，不能反映真实用户意图。

\subsection{自然语言评估集}

构造80个自然语言查询（\texttt{data/rag\_eval\_natural.json}），特征如下：

\begin{itemize}
    \item 无文档名称提示，完全模拟真实用户口语表达；
    \item 多样化表达：口语化（"健不健康"）、中英混杂（"dry run一下"）、隐晦描述；
    \item 类别分布：commands 45个，tasks 17个，safety 14个，patterns 4个。
\end{itemize}
