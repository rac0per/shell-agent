# 模拟执行与二次确认规范

## 目标
将破坏性操作转化为可预览、可确认流程，减少误删误改。

## 模拟执行原则
- 先列清单，后执行。
- 先统计数量，后确认。
- 先展示目标路径，后授权。

## 常用模拟方式
- 删除前：find ... -print
- 移动前：echo mv source target
- 批量替换前：grep 先定位命中行

## 二次确认触发条件
- 预计影响文件数超过阈值。
- 命令包含 rm、chmod、chown、mv 批量参数。
- 目标路径接近系统关键目录。

## 建议确认文案
- 将处理 N 个对象，目标路径为 XXX，是否继续。
- 检测到高风险操作，需要再次输入确认口令。

## 失败回退
- 若确认失败或超时，任务自动取消。
- 自动清理临时上下文，避免重复触发。
