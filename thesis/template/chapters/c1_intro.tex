\chapter{绪\hspace{6pt}论}

\section{研究背景与意义}

随着大语言模型（Large Language Model, LLM）能力的快速提升，以自然语言驱动Shell命令生成的智能体系统逐渐成为人机交互领域的研究热点。用户可以用口语化的中文描述操作意图，由系统自动生成对应的Shell命令并辅助执行，极大降低了Linux/Unix系统操作的门槛。

然而，通用LLM在专业Shell领域存在明显局限：其训练数据以英文为主，对中文口语化表达的理解偏差较大；对特定发行版命令、企业内部SOP流程等长尾知识的覆盖不足；且模型输出存在幻觉（Hallucination）风险，可能生成语法正确但语义错误甚至危险的命令。

检索增强生成（Retrieval-Augmented Generation, RAG）技术通过在推理时动态检索相关知识文档并注入上下文，有效缓解了上述问题。本研究在此基础上，聚焦于以下核心挑战：

\begin{enumerate}
    \item 如何构建高质量的中文Shell领域知识库，覆盖多样化的用户表达形式；
    \item 如何设计检索策略，在保证召回率的同时提升精准度和效率；
    \item 如何在系统中集成命令安全机制，防范LLM生成危险命令的风险。
\end{enumerate}

\section{研究现状}

\subsection{检索增强生成}

RAG技术由\cite{lewis2020retrieval}提出，通过将外部知识库与生成模型结合，显著提升了开放域问答任务的准确性。后续工作在检索策略、知识库构建和生成融合等方面进行了大量改进。

\subsection{混合检索策略}

纯向量检索在语义相似度上表现优异，但对精确术语的检索存在不足。BM25等稀疏检索方法在关键词匹配上更具优势。混合检索通过融合两类信号实现互补，\cite{chen2024bge}等工作验证了其在多种场景下的有效性。

\subsection{LLM安全}

Shell命令执行的安全性是部署LLM辅助系统的重要挑战。现有工作主要从提示工程和输出过滤两个角度进行防护，本文在此基础上引入了系统级的权限分级与预执行校验机制。

\section{研究内容与贡献}

本文的主要研究内容如下：

\begin{enumerate}
    \item 设计并实现了一个面向中文Shell场景的领域RAG系统，包含知识库构建、检索策略和系统集成三个核心模块；
    \item 提出了基于文档增强的词汇鸿沟缓解方法，系统性为知识库文档添加同义问法、失败查询覆盖等结构；
    \item 提出了混合检索+重排策略，并设计了关键词路由过滤机制；
    \item 构建了包含80个自然语言查询的评估集，进行了系统性的消融实验；
    \item 设计并实现了三级命令安全执行框架。
\end{enumerate}
